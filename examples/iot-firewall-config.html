🔒 Encryption flow (for zero-trust deployment)

  • Set encrypted: true in polydoc+json
  • Encrypt the config+json block content: AES-256-GCM(config, key=HMAC(device_secret, device_id))
  • Device derives key from its burned-in secret, decrypts before applying
  • Config is readable only by the intended device — even if intercepted
  • AI agents get encrypted blob; with key they can still operate on the data

📦 What is this file?

This is a PolyDoc IoT Config — a single HTML file that is simultaneously:
  • ✅ The actual device configuration (machine-readable config+json block)
  • ✅ An interactive admin panel (edit in any browser, no backend needed)
  • ✅ A versioned, signed artifact (schema version, firmware compatibility, signature)
  • ✅ AI-native — any AI agent reads config+json directly, no scraping
  • ✅ Self-contained documentation (this tab)

🚀 How to apply config to a device

Option A — HTTP POST (device has web server):
  • Click "Apply to Device" → POSTs this HTML to POST /config
  • Device parses config+json, verifies signature, applies changes, restarts services
  • ~50 lines of MicroPython / Lua / C on the device side
Option B — File upload (airgapped / offline):
  • Click "Download updated file" → get new HTML with changes baked in
  • Upload to device via SFTP, USB, or provisioning pipeline
  • Works with zero network connectivity on the management side

🤖 Minimal device HTTP handler (Python / MicroPython)

from html.parser import HTMLParser import json, hashlib class ConfigExtractor(HTMLParser): def __init__(self): super().__init__(); self.in_cfg = False; self.data = "" def handle_starttag(self, tag, attrs): if tag == "script": d = dict(attrs) if d.get("type") == "application/config+json": self.in_cfg = True def handle_endtag(self, tag): if tag == "script": self.in_cfg = False def handle_data(self, data): if self.in_cfg: self.data += data def apply_config(html_body): p = ConfigExtractor(); p.feed(html_body) cfg = json.loads(p.data) # verify signature → apply → restart services return cfg

🔄 Version compatibility check (device side)

  • Device reads polydoc+json → checks schema_version against supported range
  • Checks device.id matches own ID — rejects foreign configs
  • Checks firmware_min / firmware_max — rejects incompatible configs
  • Verifies security.signature — rejects tampered configs
  • All checks happen before any changes are applied
